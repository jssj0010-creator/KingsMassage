Netlify × GitHub 배포 체크리스트

프로젝트가 Project has not yet been deployed 로 보일 때, 한 번에 해결하는 가이드

1) 리포 루트 구조 확인

리포 최상단에 Next.js 프로젝트가 있어야 합니다. package.json, next.config.js, app/ 가 루트에 있는지 확인하세요.

✅ 올바른 루트 예시

KingMassage/
├─ app/
├─ components/
├─ package.json
├─ next.config.js
└─ netlify.toml

⛔ 서브폴더 안에 있는 경우

KingMassage/
└─ withNext.js/
   ├─ app/
   ├─ package.json
   └─ next.config.js

이 경우 Netlify의 Base directory를 withNext.js로 설정하거나, 폴더 내용을 루트로 올리세요.

2) Netlify 빌드 설정

  • Build command: npm run build
  • Publish directory: .next (Next.js App Router 기준)
  • (폴더가 하위에 있으면) Base directory: 서브폴더명

netlify.toml이 루트에 있으면 자동 인식됩니다.

[build]
command = "npm run build"
publish = ".next"

[build.environment]
NODE_VERSION = "20"
NEXT_TELEMETRY_DISABLED = "1"

3) 환경 변수

  • NODE_VERSION = 20
  • (선택) NEXT_PUBLIC_SITE_URL = Netlify 도메인 또는 사용자 도메인

로컬 .env.local에만 있고 Netlify에 없는 변수는 빌드 실패를 유발할 수 있습니다.

4) 첫 빌드 트리거

리포가 연결만 되고 커밋이 없으면 빌드가 동작하지 않습니다. 아래 중 하나로 트리거하세요.

git add -A
git commit -m "chore: initial deploy"
git push origin main

# 이미 최신이면 빈 커밋으로 강제 트리거
git commit --allow-empty -m "chore: trigger netlify build"
git push

또는 Netlify 대시보드의 Deploys 탭에서 Trigger deploy / Clear cache and deploy site 버튼 사용

5) 자주 막히는 포인트

  • 권한 문제: Netlify ↔ GitHub 권한이 해당 리포에 부여되어 있는지 확인
  • Node 버전: 로그에 16/18이 보이면 환경 변수 NODE_VERSION=20 설정 후 재빌드
  • 잘못된 Publish 디렉터리: App Router는 .next (Static Export가 아니면 out 아님)
  • 패키지 설치 실패: 일시적으로 Build command를 npm i && npm run build로 변경
  • 루트에 package.json 없음: 루트로 이동하거나 Base directory 설정

6) CLI로 강제 배포(선택)

npm i -g netlify-cli
netlify login
netlify init
netlify env:set NODE_VERSION 20
netlify deploy --build --prod

7) Next.js 품질 체크 팁

  • next/image로 이미지 최적화, 내부 링크는 next/link 사용
  • 상수(지역 배열)는 파일 분리, 데이터 페칭은 revalidate 또는 On-Demand Revalidation
  • 무거운 컴포넌트는 dynamic(..., { ssr:false })로 지연 로딩
